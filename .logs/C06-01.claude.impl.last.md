### Risks / Follow-ups

1. **WordPress test suite required**: The integration tests cannot run without the WordPress test suite installed. The PHP tests require setting up `WP_TESTS_DIR` environment variable

2. **Container availability**: Tests now depend on the Plugin container being properly initialized. If the container is not available (e.g., in isolation tests), the tests will fail

3. **Settings caching**: The `SettingsRegistry` has internal caching. Since we're calling `update_option()` directly in tests, the cached values in the `SettingsInterface` instance may not reflect the new values immediately. This could cause `test_rate_limit_setting()` to fail if the settings instance doesn't pick up the new value. This is a pre-existing design consideration in the original tests that may need review
